# Gmail連携メール一斉送信システム

# Use Case Specification（ユースケース仕様書）

**対象SRS：Bulk Email Sending Platform**

---

# 1. システム概要

本システムは、ユーザーが自身の Gmail アカウントを連携し、本人アドレスのみを送信元として利用できる一斉送信プラットフォームである。

---

# 2. アクター定義

| アクター                | 説明                      |
| ------------------- | ----------------------- |
| 一般ユーザー              | Gmail連携し、メール送信・受信を行う利用者 |
| 管理者                 | システム監視・制限設定・ユーザー管理      |
| Google OAuth Server | 認証提供                    |
| Gmail API           | メール送受信処理                |
| Scheduler           | 予約送信実行                  |
| Java Mail Engine    | 高速送信ワーカー                |

# 4. 詳細ユースケース

---

# UC-01 ユーザー登録

## 目的

新規利用者がアカウント作成する。

## アクター

一般ユーザー

## 事前条件

未登録であること

## 基本フロー

1. ユーザーが登録画面表示
2. メールアドレス入力
3. パスワード入力
4. 利用規約同意
5. 登録実行
6. システムが保存
7. 完了通知

## 例外

* 既存メールアドレス → 登録拒否
* パスワード弱い → 再入力

---

# UC-03 Gmailアカウント連携

## 目的

ユーザー自身のGmailを接続する。

## アクター

一般ユーザー
Google OAuth Server

## 基本フロー

1. ユーザーが「Gmail連携」押下
2. Google認証画面へ遷移
3. ログイン
4. 権限許可
5. OAuth code返却
6. システムがtoken取得
7. Gmailアドレス保存
8. 接続完了表示

## 例外

* 認証拒否
* Token取得失敗
* API制限超過

---

# UC-04 宛先CSV登録

## 目的

送信先リストを一括登録する。

## アクター

一般ユーザー

## 基本フロー

1. CSVアップロード
2. システムが形式検証
3. メール形式チェック
4. 重複除去
5. DB保存
6. 登録件数表示

## 例外

* CSV形式不正
* アドレス不正
* 上限件数超過

---

# UC-06 テンプレート作成

## 目的

送信本文を保存する。

## 基本フロー

1. 件名入力
2. 本文入力
3. HTML/テキスト選択
4. 差し込み変数設定（{{name}} 等）
5. 保存

---

# UC-07 一斉送信実行（主要UC）

## 目的

複数宛先へ一括送信する。

## アクター

* 一般ユーザー
* Java Mail Engine
* Gmail API

## 事前条件

* Gmail連携済み
* 宛先あり
* テンプレートあり

## 基本フロー

1. ユーザーが送信画面表示
2. 宛先グループ選択
3. テンプレート選択
4. 添付指定（任意）
5. 即時送信押下
6. Python側が送信ジョブ生成
7. Java Mail Engineへ投入
8. Java側で並列送信開始
9. Gmail API送信
10. 成否ログ保存
11. 完了通知

## 例外

* Gmail token失効
* API Rate Limit
* 宛先不正
* 添付容量超過
* ネットワーク障害

---

# UC-08 予約送信

## 目的

指定日時に送信する。

## アクター

一般ユーザー
Scheduler

## 基本フロー

1. 配信内容作成
2. 日時指定
3. 保存
4. Scheduler待機
5. 指定時刻到達
6. UC-07起動

---

# UC-09 送信履歴確認

## 目的

送信結果を確認する。

## 基本フロー

1. 履歴画面表示
2. キャンペーン一覧表示
3. 件数・成功率表示
4. 詳細表示

---

# UC-10 Gmail受信確認

## 目的

受信メール確認

## アクター

一般ユーザー
Gmail API

## 基本フロー

1. 受信箱表示
2. Gmail API取得
3. 件名一覧表示
4. スレッド閲覧

---

# UC-11 配信停止処理

## 目的

受信者が解除する。

## アクター

受信者

## 基本フロー

1. メール下部リンク押下
2. 停止画面表示
3. 確定
4. blacklist登録
5. 今後自動除外

---

# UC-13 ユーザー停止（管理者）

## 目的

不正利用停止

## アクター

管理者

## 基本フロー

1. 管理画面表示
2. ユーザー検索
3. 停止実行
4. ログイン不可化
5. Gmail送信停止

---

# UC-15 エラー再送

## 目的

失敗分のみ再送

## 基本フロー

1. 履歴画面表示
2. failed抽出
3. 再送押下
4. 新規ジョブ生成
5. 再送実行
