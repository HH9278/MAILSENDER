## 1. Introduction

### 1.1 Purpose

本書は、Gmailアカウント連携型メール一斉送信システムの機能要件・非機能要件・制約条件を定義する。

### 1.2 Scope

本システムは、ユーザー自身の Gmail アカウントを利用し、本人アドレスのみを送信元として、一斉送信・予約送信・履歴管理・受信確認を行うWebベースシステムである。

## 3. Specific Requirements

---

## 3.1 Functional Requirements

### FR-01 User Registration

システムはユーザー登録機能を提供しなければならない。

### FR-02 Authentication

ログイン / ログアウト機能を提供しなければならない。

### FR-03 Gmail Integration

OAuth2による Gmail 接続機能を提供しなければならない。

### FR-04 Sender Restriction

認証済み本人Gmailアドレスのみ送信元として利用可能でなければならない。

### FR-05 Contact Management

CSV登録・編集・削除機能を提供しなければならない。

### FR-06 Template Management

メールテンプレート保存機能を提供しなければならない。

### FR-07 Bulk Email Sending

複数宛先への一括送信機能を提供しなければならない。

### FR-08 Scheduled Sending

予約送信機能を提供しなければならない。

### FR-09 Receiving Mail

受信一覧取得機能を提供しなければならない。

### FR-10 Delivery History

送信履歴を保持し表示しなければならない。

### FR-11 Unsubscribe

配信停止機能を提供しなければならない。

### FR-12 Admin Control

管理者はユーザー停止・制限設定を行えること。

## UC-07 Bulk Email Send

### Use Case ID

UC-07

### Use Case Name

Bulk Email Send

### Primary Actor

Standard User

### Supporting Actors

・Gmail API
・Java Mail Engine

### Goal

複数宛先へ一括メール送信する。

### Preconditions

・User authenticated
・Gmail connected
・Contact list exists

### Trigger

ユーザーが送信ボタン押下

### Main Success Scenario

1. User selects recipients
2. User selects template
3. User clicks Send
4. System creates campaign
5. Python backend creates job
6. Java engine dispatches emails
7. Gmail API sends messages
8. Results stored
9. Completion displayed

### Extensions

#### 6a Rate Limit Hit

ジョブ待機後再試行

#### 7a Invalid Recipient

該当宛先失敗記録

#### 7b Token Expired

再認証要求

### Postconditions

・Send history stored
・Success / failure counted

### Business Rules

・Sender must equal authenticated Gmail account
・Unsubscribed recipients excluded
