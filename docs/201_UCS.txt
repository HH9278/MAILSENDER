# Gmail連携メール一斉送信システム

# Use Case Specification（改訂版）

対象SRS：Bulk Email Sending Platform
反映レビュー：総務課長 / 一般事務 / SE

---

# 1. システム概要

本システムは、ユーザーが自身の Gmail アカウントを連携し、本人アドレスのみを送信元として利用できる一斉送信プラットフォームである。
加えて、企業利用を想定し、承認統制・誤送信防止・高可用性・現場操作性を備える。

---

# 2. アクター定義

一般ユーザー		メール作成・送信・受信を行う利用者
承認者			送信内容を確認し承認する上長・責任者
管理者			システム管理・制限設定・監査対応
Google OAuth Server	認証提供
Gmail API		メール送受信処理
Scheduler		予約送信実行
Java Mail Engine	高速送信ワーカー
監視システム		障害検知・通知

# UC-07 一斉送信実行（主要UC）

## 目的

複数宛先へ安全に一括送信する。

## アクター

一般ユーザー
承認者
Java Mail Engine
Gmail API

## 事前条件

・Gmail連携済み
・宛先あり
・テンプレートあり
・必要時は承認済み

## 基本フロー

1. ユーザーが送信画面表示
2. 宛先グループ選択
3. テンプレート選択
4. 添付指定（任意）
5. UC-19 送信前チェック実行
6. 必要時 UC-17 承認依頼
7. 承認完了
8. 即時送信押下
9. Python側が送信ジョブ生成
10. UC-23 重複送信チェック
11. Java Mail Engineへ投入
12. Java側で並列送信開始
13. Gmail API送信
14. 成否ログ保存
15. 完了通知

## 例外

・Gmail token失効 → UC-24
・API Rate Limit → 自動待機再試行
・宛先不正 → 該当宛先除外
・添付容量超過 → 中断通知
・ネットワーク障害 → 再試行
